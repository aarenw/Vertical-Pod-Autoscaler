# `create-vpa.sh` Script Usage Guide
[中文版本](create-vpa-guide.md)

This script aims to simplify the process of creating Vertical Pod Autoscalers (VPA) in an OpenShift/Kubernetes cluster. It can automatically generate and apply VPA resources for all `Deployment`, `StatefulSet`, and `DaemonSet` in a specified namespace.

## Script Purpose

*   **Automate VPA Creation**: Batch create VPAs for various workload types (Deployment, StatefulSet, DaemonSet).
*   **Flexible Update Modes**: Supports multiple VPA update strategies, defaulting to `Off` mode, which only provides resource recommendations without automatically modifying Pods.
*   **Local YAML Saving**: Allows saving generated VPA YAML files to a local directory for review or manual application.

## Usage

```bash
./create-vpa.sh -n <namespace> [-m <UpdateMode>] [-o <output-directory>]
```

### Parameter Description

*   `-n` or `--namespace` (Required)
    *   **Description**: Specifies the target OpenShift/Kubernetes namespace where VPA resources are to be created.
    *   **Example**: `-n my-project`

*   `-m` 或 `--mode` (Optional)
    *   **Description**: Defines the VPA`s resource update strategy. This directly maps to the `spec.updatePolicy.updateMode` field of the VPA resource.
    *   **Optional Values**:
        *   `Off` (Default): VPA only calculates and provides resource recommendations, but does not make any modifications to Pods. **Recommended for initial use or observation phases.**
        *   `Auto`: VPA automatically adjusts Pod resource requests and limits during runtime based on recommendations. This may lead to Pod restarts to apply new resource configurations.
        *   `Initial`: VPA only applies recommended resource requests and limits when the Pod is first created, with no further automatic updates thereafter.
    *   **Example**: `-m Auto`

*   `-o` or `--output` (Optional)
    *   **Description**: Specifies a local directory path. If provided, the script will save all generated VPA YAML files to this directory instead of directly applying them to the cluster. This is useful for reviewing VPA configurations before application.
    *   **Example**: `-o ./vpa-configs`

## Examples

1.  **Create VPA for all workloads in the `my-project` namespace (default `Off` mode):**
    ```bash
    ./create-vpa.sh -n my-project
    ```

2.  **Create VPA in `Auto` mode for workloads in the `my-project` namespace and save YAML files to the `./vpa-configs` directory:**
    ```bash
    ./create-vpa.sh -n my-project -m Auto -o ./vpa-configs
    ```

3.  **Create VPA in `Initial` mode for workloads in the `production` namespace:**
    ```bash
    ./create-vpa.sh -n production -m Initial
    ```

## Notes

*   **CLI Tools**: The script relies on `oc` (OpenShift CLI) or `kubectl` (Kubernetes CLI) commands. Ensure that one of these tools is installed and configured in your environment.
*   **Permissions**: The user running the script needs permissions to create and list `Deployment`, `StatefulSet`, `DaemonSet`, and `VerticalPodAutoscaler` resources in the target namespace.
*   **Resource Limits**: The VPA generated by the script defaults to including `resourcePolicy`, setting a recommended range of `cpu: 50m - 2` and `memory: 64Mi - 2Gi` for containers. You can adjust these default values as needed.
